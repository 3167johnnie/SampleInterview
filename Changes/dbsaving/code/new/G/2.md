1. languages is not available to ConsentPopup.jsp

Your <s:iterator value="languages"> only works if the JSP can access HomeLoanAction.getLanguages().

Best fix: load languages in getHomeLoan() before any JSP return.

private void loadPrivacyLanguages() {
	try {
		languages = commonService.getAllActiveLanguages();
		logger.info("Privacy language dropdown count : "
				+ (languages == null ? 0 : languages.size()));
	} catch (Exception e) {
		logger.info("Exception in loadPrivacyLanguages", e);
	}
}

Then in getHomeLoan() add near top:

public String getHomeLoan(Integer moduleId){
	logger.info("inside getHomeLoan start...");
	loadPrivacyLanguages();
	loanTypeId=Constants.HOME_LOAN_ID;

This is safer than only adding it inside generateUIBeanList().

2. Spring bean injection may be missing

If this is missing, masterLanguageDao will be null.

In Spring XML, make sure you have:

<bean id="masterLanguageDao"
	class="com.mintstreet.consent.dao.MasterLanguageDao">
</bean>

Inside commonService bean:

<property name="masterLanguageDao" ref="masterLanguageDao" />

Right click page → View Page Source / Inspect.

If you see this:

then JSP did not get languages.

If you see:

then entity getter names or OGNL expression is wrong.

If you see correct options but dropdown visually blank, then CSS issue.

6. CSS may hide option text

Your select and parent div both have same class:

<div class ="privacy-consent-dropdown">
<select id="privacyLocaleDropdown" class="privacy-consent-dropdown">

If CSS is bad, it may hide text.

Test once with simple class:

<select id="privacyLocaleDropdown"
	class="form-control"
	onchange="loadPrivacyByLocale();">

If it shows options, your CSS class is causing blank dropdown.

7. Safer fallback: use AJAX dropdown

If JSP iterator still fails, use AJAX population. That works even when JSP cannot access languages.

Keep HTML empty:

<select id="privacyLocaleDropdown"
	class="privacy-consent-dropdown"
	onchange="loadPrivacyByLocale();">
</select>

On modal open:

$(document).on("shown.bs.modal", "#consentHomeLoan", function(){
	loadPrivacyLanguageDropdown();
});

Use function:

function loadPrivacyLanguageDropdown(){
	$.ajax({
		url : "getPrivacyLanguageList",
		type : "POST",
		success : function(response){
			var json = typeof response === "string"
				? JSON.parse(response)
				: response;
			console.log("Language Response:", json);
			var optionHtml = "";
			if(json.status == "success" && json.languageList.length > 0){
				$.each(json.languageList, function(index, item){
					optionHtml += "<option value='" + item.locale + "'>"
						+ item.languageName
						+ "</option>";
				});
				$("#privacyLocaleDropdown").html(optionHtml);
				$("#privacyLocaleDropdown").val("eng");
				loadPrivacyByLocale("eng");
			}
		}
	});
}

My recommendation: for your project, AJAX dropdown is safer because your popup JSP may be included inside another JSP and may not directly access HomeLoanAction.languages.
